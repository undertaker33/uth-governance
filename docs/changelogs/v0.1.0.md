# v0.1.0：UTH Governance 初始发布

发布日期：2026-05-16

## 新增

- 发布全局安装流程与 `uth-onboarding` 项目接管边界，明确区分“安装治理包”和“启用目标项目”。
- 提供 `uth-governance`、`uth-dev`、`uth-debug`、`uth-review`、`uth-docs`、`uth-git` 等场景 skills，以及配套的 `uth-sp-*` 方法 skills。
- 提供参考 Hook runner `tools/uth-hooks/`，覆盖 L0/L1/L2/L3 路由、写入、Git、UTF-8 与收口门禁，并附带验证脚本与单元测试。

## 改进

- 明确 Hook 门禁文档中的“当前已实现”与“仍待实现或硬化”边界，避免把未落地 gate 写成现状。
- `uth-git` 增补 Codex App Git directive 约束，要求 UI 指令中的 `cwd` 使用正斜杠路径。

## 修复

- 修正文档对 `Context-source`、`Archive`、`Current-state`、`UTH-SP evidence` 等 gate 的实现状态描述，使其与当前 runner 行为一致。

## 需要注意

- 本版本是治理包的初始正式版本；内部手册与模板文件名仍不带版本后缀，版本号以 release artifact 和 Git tag 为准。
- `Context-source` 基础收口门禁已实现；`UTH-SP-required-evidence`、`Archive-read`、`Current-state-staleness` 仍待后续实现或硬化。
