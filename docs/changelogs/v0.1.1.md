# v0.1.1：正式任务与文档预检门禁硬化

发布日期：2026-05-21

## 新增

- L1 Process Gate 对正式开发增加任务包证据要求：`formal-dev` / `todo-implementation` 必须提供 active task package、accepted Design、current Todo；`todo-breakdown` / `handoff-from-design` 必须提供 active task package 和 accepted Design。
- `uth-design -> uth-dev` 切换在 `explicit_handoff` 之外新增用户确认门禁，缺少确认时返回 `ASK design-dev-user-confirmation-missing`。
- 治理 Markdown 写入前新增 `document_preflight` 预检，要求先完成 `uth-sp-brainstorming` 并确认没有开放用户问题；L1 写入意图和 L2 文件写入都会检查。
- Web 页面、Web app screen、browser UI、frontend page layout 设计需要 `ui-ux-pro-max` 证据；Android UI/UX 设计不触发该要求。

## 改进

- 同步更新 `uth-dev`、`uth-design`、`uth-docs`、`uth-debug`、`uth-review`、`uth-git`、`uth-onboarding`、`uth-governance` 与相关 UTH-SP 技能说明。
- 更新 `README.md`、工程治理启动手册、HOOKS 门禁手册和全链路流程图，使说明与 hook 行为一致。
- 同步 `skills/uth-onboarding/assets/uth-hooks/`，确保目标项目 onboarding 后复制到的 hook 与参考实现一致。

## 验证

- `python scripts\verify.py`
- `PYTHONUTF8=1` 下对变更技能运行 `quick_validate.py`
- `python skills\uth-utf8-guard\scripts\check_utf8_docs.py ...`

## 风险与回滚

- 调用方或 wrapper 需要补充 `document_preflight`、正式任务包证据和 `ui_ux_pro_max` 字段，否则新门禁会按预期阻断。
- 如果需要回滚，可回退本版本提交；本次提交不创建 tag 或 release artifact。
